Aufgabenplanung

Die Aufgabenplanung dient zur Einsicht und Steuerung der im System hinterlegten geplanten Hintergrundprozesse. Diese Prozesse werden in regelmässigen Intervallen automatisch ausgeführt (z. B. Datensynchronisationen, Prüfungen, Backups).

Geplante Aufgaben

Im oberen Bereich werden alle aktuell geplanten Aufgaben angezeigt, die über den internen Scheduler konfiguriert sind. Zu jedem Eintrag werden folgende Informationen dargestellt:

  • Befehl – Name des Artisan Commands
  • Beschreibung – Kurzbeschreibung des Befehls
  • Intervall – menschlich lesbarer Ausführungsrhythmus (z. B. „Täglich um 02:00 Uhr“)
  • Nächste Ausführung – geplanter Zeitpunkt der kommenden Ausführung

Die Sortierung erfolgt automatisch nach dem nächsten Ausführungstermin, damit zeitnahe Tasks oben erscheinen.

Manuell ausführen

Unterhalb der Übersicht befinden sich Befehle, die manuell gestartet werden können. Jeder Eintrag zeigt:

  • den Befehlsnamen
  • eine Beschreibung
  • einen Button zum Ausführen

Beim Klick auf Ausführen wird der entsprechende Artisan Command direkt im Backend gestartet. Während der Ausführung wird der Button deaktiviert, um parallele Starts zu verhindern.

Ausgabeanzeige

Nach dem Start erscheint automatisch ein Dialogfenster, das folgende Informationen enthält:

  • gestarteter Befehl
  • Startzeitpunkt
  • Dauer der Ausführung
  • Konsolenausgabe des Commands

Fehler werden ebenfalls direkt im Ausgabedialog angezeigt, einschliesslich der originalen Fehlermeldung.

Zweck

Die Aufgabenplanung dient zur Überwachung, Diagnose und manuellen Kontrolle von automatisierten Prozessen. Sie ermöglicht sowohl das Prüfen der nächsten geplanten Ausführung als auch das gezielte manuelle Ausführen einzelner Tasks – beispielsweise zur Sofortaktualisierung von Daten.
