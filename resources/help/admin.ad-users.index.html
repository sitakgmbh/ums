Active-Directory Benutzerübersicht

Diese Seite zeigt alle im System bekannten Active-Directory Benutzer in Tabellenform an. Über Filter, Suche und Aktionen können Datensätze eingegrenzt, analysiert und exportiert werden.

Tabelle

Die Tabelle stellt grundlegende Identifikationsdaten sowie Statusinformationen dar. Die folgenden Spalten werden angezeigt:

  • Anzeigename – kombinierter Anzeigename des Benutzers.
  • Vorname / Nachname – Stammdaten aus dem AD.
  • Benutzername – Kürzel des AD-Kontos.
  • Account-Status – zeigt, ob das Konto aktiviert oder deaktiviert ist.
  • AD-Status – zeigt, ob der Benutzer im AD vorhanden oder gelöscht wurde.
  • Aktionen – ermöglicht den Aufruf der Detailansicht.

Sortierung

Die Tabelle kann über klickbare Spalten sortiert werden. Standardmäßig wird nach dem Anzeigenamen aufsteigend sortiert.

Suche

Über das Suchfeld können Benutzer nach Name oder Benutzername gefiltert werden. Die Suche erfolgt in folgenden Feldern:

  • Anzeigename
  • Vorname
  • Nachname
  • Benutzername

Filter

Zusätzlich zur Suche stehen spezielle Filter zur Verfügung:

  • Deaktivierte Benutzer anzeigen/ausblenden – betrifft den Account-Status.
  • Gelöschte Benutzer anzeigen/ausblenden – blendet im AD nicht mehr vorhandene Konten aus.

Änderungen an den Filtern aktualisieren die Tabelle unmittelbar.

Export

Über Export CSV kann der aktuelle Tabellenstand in eine CSV-Datei exportiert werden. Dabei werden die aktuell sichtbaren Daten berücksichtigt.

Detailansicht

Über die Aktion kann ein Benutzer geöffnet werden. In der Detailansicht stehen zusätzliche Funktionen wie Passwort-Reset oder Statusänderungen zur Verfügung.

Modale Fenster

Die Übersichtsseite verfügt über zusätzliche Analysefunktionen, die situationsbezogen über Modalfenster dargestellt werden.

Geburtstage

Das Modal Geburtstage zeigt Benutzer mit anstehenden Geburtstagen an. Es stehen folgende Filter zur Verfügung:

  • Heute – Benutzer, die genau heute Geburtstag haben.
  • Diese Woche – Benutzer mit Geburtstag innerhalb der nächsten 7 Tage.
  • Diesen Monat – Benutzer im aktuellen Monat.

Angezeigt werden Name, Geburtstag und verbleibende Tage bis zum nächsten Termin.

SAP ↔ AD Matching

Das Modal SAP ↔ AD stellt einen Abgleich der Personalinformationen zwischen SAP und Active Directory dar. Über die Filter können Unstimmigkeiten selektiv analysiert werden:

  • Keine Personalnummer – AD-Benutzer ohne verknüpfte Personalnummer.
  • Kein SAP Eintrag – Personalnummer existiert im AD, aber nicht im SAP.
  • Kein AD Benutzer – Eintrag in SAP ohne entsprechendes AD-Konto.

Das Modal dient zur Überwachung der Datenqualität und zur Klärung von Zuordnungsproblemen.

Zweck

Die Übersicht dient als zentrale Verwaltungs- und Kontrollstelle für AD-Benutzerkonten und unterstützt sowohl administrative Tätigkeiten als auch Plausibilitätskontrollen gegen SAP und KIS.
