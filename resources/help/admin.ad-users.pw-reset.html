Passwort-Reset (AD / KIS)

Auf dieser Seite können AD- und KIS-Benutzerkonten administriert und Passwörter zurückgesetzt werden.

Active Directory

Im Bereich Active Directory wird der Benutzer anhand seines AD-Benutzernamens geladen. Der Benutzername wird automatisch vorbelegt und kann nicht verändert werden.

Neues Passwort

Es kann ein Passwort manuell eingegeben oder über den Button automatisch generiert werden. Beim Speichern wird das Passwort im AD gesetzt.

Kontozustand

Folgende Eigenschaften des Kontos können bearbeitet werden:

  • Account entsperren – hebt eine vorhandene AD-Sperre auf.
  • Account aktivieren/deaktivieren – schaltet das Konto im AD frei oder deaktiviert es.
  • Passwort bei nächster Anmeldung ändern – setzt bzw. entfernt die AD-Flag für erzwungene Passwortänderung.

Anpassungen im AD können auch ohne Passwortänderung gespeichert werden. Teilanpassungen (z. B. nur entsperren oder nur aktivieren/deaktivieren) sind möglich und werden unmittelbar auf das AD angewendet.

Nach dem Speichern werden Statusinformationen erneut aus dem AD geladen und im Formular aktualisiert.

KIS (ORBIS)

Im unteren Bereich können die entsprechenden KIS-Daten des Benutzers bearbeitet werden. Dafür muss zunächst das Kürzel im Feld Benutzername gesucht werden.

Benutzer finden

Durch Klick auf wird der Benutzer im KIS gesucht. Nach erfolgreicher Suche erscheint der aktuelle Kontostatus.

Neues Passwort und Einstellungen

Das neue Passwort kann eingegeben oder ebenfalls automatisch generiert werden. Beim Speichern wird das Passwort sofort in ORBIS gesetzt.

Im KIS können darüber hinaus folgende Zustände gesetzt werden:

  • Entsperrt / Gesperrt
  • Passwort bei nächster Anmeldung ändern

Im Gegensatz zum AD ist im KIS keine reine Teilanpassung möglich. Für eine Änderung muss immer ein neues Passwort gesetzt werden. Ohne Passwortänderung werden keine weiteren Einstellungen übernommen.

Nach erfolgreichem Speichern wird der KIS-Datensatz erneut geladen, um den aktuellen Zustand anzuzeigen.
